  <p>एक <a href="/reference/p5/p5.Geometry">p5.Geometry</a> ऑब्जेक्ट बनाने के लिए एक 3D मॉडल लोड करता है।</p>

  <p><code>loadModel()</code> OBJ और STL फ़ाइलों से 3D मॉडल लोड कर सकता है। एक बार मॉडल लोड हो जाने पर, इसे <a href="/reference/p5/model">model()</a> फ़ंक्शन के साथ प्रदर्शित किया जा सकता है, जैसा कि <code>model(shape)</code> में है।</ पी>

  <p>मॉडल को संसाधित करने में सहायता के लिए वैकल्पिक पैरामीटर के साथ <code>loadModel()</code> को कॉल करने के तीन तरीके हैं।</p>

  <p>पहला पैरामीटर, <code>path</code>, हमेशा फ़ाइल के पथ के साथ एक <code>string</code> होता है। स्थानीय फ़ाइलों के पथ सापेक्ष होने चाहिए, जैसे <code>loadModel('/assets/model.obj')।

  </code>'<a href='https://example.com/model.obj'%60'>https://example.com/model.obj'`</a> जैसे यूआरएल को ब्लॉक किया जा सकता है ब्राउज़र सुरक्षा के लिए।</p>

  <p>कॉल करने का पहला तरीका <code>loadModel()</code> में फ़ाइल पथ के बाद तीन वैकल्पिक पैरामीटर हैं।

  पहला वैकल्पिक पैरामीटर, <code>successCallback</code>, मॉडल लोड होने के बाद कॉल करने के लिए एक फ़ंक्शन है। उदाहरण के लिए, <code>loadModel('/assets/model.obj', HandleModel)</code> मॉडल लोड होने के बाद <code>handleModel()</code> फ़ंक्शन को कॉल करेगा।

  दूसरा वैकल्पिक पैरामीटर, <code>failureCallback</code>, मॉडल लोड होने में विफल होने पर कॉल करने के लिए एक फ़ंक्शन है। उदाहरण के लिए, यदि लोड करते समय कोई त्रुटि होती है, तो <code>loadModel('/assets/model.obj', HandleModel, HandleFairure)</code> <code>handleFairure()</code> फ़ंक्शन को कॉल करेगा।

  तीसरा वैकल्पिक पैरामीटर, <code>fileType</code>, एक स्ट्रिंग के रूप में मॉडल का फ़ाइल एक्सटेंशन है। उदाहरण के लिए, <code>loadModel('/assets/model', हैंडलमॉडल, हैंडलफेल्योर, '.obj')</code> फ़ाइल मॉडल को <code>.obj</code> फ़ाइल के रूप में लोड करने का प्रयास करेगा।</code> पी>

  <p>कॉल करने का दूसरा तरीका <code>loadModel()</code> में फ़ाइल पथ के बाद चार वैकल्पिक पैरामीटर हैं।

  पहला वैकल्पिक पैरामीटर एक <code>boolean</code> मान है। यदि <code>true</code> पास हो जाता है, जैसे कि <code>loadModel('/assets/model.obj', true)</code>, तो यह सुनिश्चित करने के लिए मॉडल का आकार बदल दिया जाएगा कि यह कैनवास पर फिट बैठता है।

  अगले तीन पैरामीटर हैं <code>successCallback</code>, <code>failureCallback</code>, और <code>fileType</code> जैसा कि ऊपर बताया गया है।</p>

  <p>कॉल करने का तीसरा तरीका <code>loadModel()</code> में फ़ाइल पथ के बाद एक वैकल्पिक पैरामीटर है। वैकल्पिक पैरामीटर, <code>options</code>, विकल्पों के साथ एक <code>object</code> है, जैसे <code>loadModel('/assets/model.obj', options)</code> में। <code>options</code> ऑब्जेक्ट में निम्नलिखित गुण हो सकते हैं:</p>

  <code>

  let options = {
    // Enables standardized size scaling during loading if set to true.
    normalize: true,

  <p>  // Function to call once the model loads.
    successCallback: handleModel,</p>
  <p>  // Function to call if an error occurs while loading.
    failureCallback: handleError,</p>
  <p>  // Model's file extension.
    fileType: '.stl',</p>
  <p>  // Flips the U texture coordinates of the model.
    flipU: false,</p>
  <p>  // Flips the V texture coordinates of the model.
    flipV: false
  };</p>

  <p>// Pass the options object to loadModel().

  loadModel('/assets/model.obj', options);

  </p></code>

  <p>मॉडल को लोड होने में समय लग सकता है। <code>loadModel()</code> को <a href='/reference/p5/preload'>preload()</a> में कॉल करने से यह सुनिश्चित होता है कि मॉडल <a href='/reference/p5/setup">setup()</a> या <a href="/reference/p5/draw">draw()</a> इस्तेमाल होने से पहले लोड हो जाएं।</p> 

  <p>ध्यान दें: रंगीन एसटीएल फ़ाइलों के लिए कोई समर्थन नहीं है। रंग वाली एसटीएल फ़ाइलें बिना रंग के प्रस्तुत की जाएंगी।</p>
