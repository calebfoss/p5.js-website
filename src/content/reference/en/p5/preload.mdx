  <p>Called directly before <a href="/reference/p5/setup">setup()</a>, the <a
  href="/reference/p5/preload">preload()</a> function is used to handle

  asynchronous loading of external files in a blocking way. If a preload

  function is defined, <a href="/reference/p5/setup">setup()</a> will wait until
  any load calls within have

  finished. Nothing besides load calls (<a
  href="/reference/p5/loadImage">loadImage</a>, <a
  href="/reference/p5/loadJSON">loadJSON</a>, <a
  href="/reference/p5/loadFont">loadFont</a>,

  <a href="/reference/p5/loadStrings">loadStrings</a>, etc.) should be inside
  the preload function. If asynchronous

  loading is preferred, the load methods can instead be called in <a
  href="/reference/p5/setup">setup()</a>

  or anywhere else with the use of a callback parameter.</p>

  <p>By default the text "loading..." will be displayed. To make your own

  loading page, include an HTML element with id "p5_loading" in your

  page. More information <a href="http://bit.ly/2kQ6Nio">here</a>.</p>
