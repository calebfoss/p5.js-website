  <p>코드 블록을 실행할지 여부를 선택하는 방법입니다.</p>

  <p><code>if</code> 문은 조건에 따라 코드 블록을 실행하는 데 도움이 됩니다. 예를 들어, <code>if</code>문을 사용하면 "마우스를 눌렀을 때 원을 그리기"라는 아이디어를 쉽게 표현할 수 있습니다:</p>

  <code>

  if (mouseIsPressed === true) {
    circle(mouseX, mouseY, 20);
  }

  </code>


  <p>문장 헤더는 키워드 <code>if</code>로 시작합니다. 괄호 안의 표현식 <code>mouseIsPressed === true</code> 는 <code>true</code> 또는 <code>false</code>가 될 수 있는 <code>Boolean</code> 표현식입니다.
  expression that's

  either <code>true</code> or <code>false</code>. The code between the curly
  braces <code>{}</code> is the if

  statement's body. The body only runs if the <code>Boolean</code> expression is
  <code>true</code>.</p>

  <p>The <a href="/reference/p5/mouseIsPressed">mouseIsPressed</a> system
  variable is

  always <code>true</code> or <code>false</code>, so the code snippet above
  could also be written

  as follows:</p>

  <code>

  if (mouseIsPressed) {
    circle(mouseX, mouseY, 20);
  }

  </code>


  <p>An <code>if</code>-<code>else</code> statement adds a block of code that
  runs if the <code>Boolean</code>

  expression is <code>false</code>. For example, here's an
  <code>if</code>-<code>else</code> statement that

  expresses the idea "Draw a circle if the mouse is pressed. Otherwise,

  display a message.":</p>

  <code>

  if (mouseIsPressed === true) {
    // When true.
    circle(mouseX, mouseY, 20);
  } else {
    // When false.
    text('Click me!', 50, 50);
  }

  </code>


  <p>There are two possible paths, or branches, in this code snippet. The

  program must follow one branch or the other.</p>

  <p>An <code>else</code>-<code>if</code> statement makes it possible to add
  more branches.

  <code>else</code>-<code>if</code> statements run different blocks of code
  under different

  conditions. For example, an <code>else</code>-<code>if</code> statement makes
  it easy to express

  the idea "If the mouse is on the left, paint the canvas white. If the mouse

  is in the middle, paint the canvas gray. Otherwise, paint the canvas

  black.":</p>

  <code>

  if (mouseX < 33) {
    background(255);
  } else if (mouseX < 67) {
    background(200);
  } else {
    background(0);
  }

  </code>


  <p><code>if</code> statements can add as many
  <code>else</code>-<code>if</code> statements as needed. However,

  there can only be one <code>else</code> statement and it must be last.</p>

  <p><code>if</code> statements can also check for multiple conditions at once.
  For

  example, the <code>Boolean</code> operator <code>&&</code> (AND) checks
  whether two expressions

  are both <code>true</code>:</p>

  <code>

  if (keyIsPressed === true && key === 'p') {
    text('You pressed the "p" key!', 50, 50);
  }

  </code>


  <p>If the user is pressing a key AND that key is <code>'p'</code>, then a
  message will

  display.</p>

  <p>The <code>Boolean</code> operator <code>||</code> (OR) checks whether at
  least one of two

  expressions is <code>true</code>:</p>

  <code>

  if (keyIsPressed === true || mouseIsPressed === true) {
    text('You did something!', 50, 50);
  }

  </code>


  <p>If the user presses a key, or presses a mouse button, or both, then a

  message will display.</p>

  <p>The body of an <code>if</code> statement can contain another
  <code>if</code> statement. This is

  called a "nested <code>if</code> statement." For example, nested
  <code>if</code> statements make

  it easy to express the idea "If a key is pressed, then check if the key is

  'r'. If it is, then set the fill to red.":</p>

  <code>

  if (keyIsPressed === true) {
    if (key === 'r') {
      fill('red');
    }
  }

  </code>


  <p>See <a href="/reference/p5/Boolean">Boolean</a> and <a
  href="/reference/p5/Number">Number</a>

  to learn more about these data types and the operations they support.</p>
